# Atoms Demo 架构设计

## 产品主线

界面由公开 Landing、登录后的 Home 和“对话 + Reviewer”三层组成，不要求用户先选择模式。项目列表、Console、Files、Design 和可编辑 Code 是检查与修改面；意图、澄清、计划、工具执行、验证与增量修改都回到同一条对话时间线。

## Agent runtime

```text
用户输入
  ↓
识别 intent + domain ── 关键输入不足 ──→ 结构化澄清 ──→ 恢复规划
  ↓
选择必要专家与受控工具，生成依赖计划
  ↓
人工批准
  ↓
执行 tool → 写入共享 artifact → 发出流式事件
  ↓
规范化 UI schema + appType → 生成 App.jsx / styles.css / app.config.json
  ↓
validate ── 失败 ──→ 单次 repair ──→ 再验证
  ↓
App Viewer + Working Process
```

模块职责：

- `agent/intent.mjs`：意图、领域、置信度、特征和澄清判断。
- `agent/prompts.mjs`：模型输出协议；约束修改任务以已有 preview 为基线。
- `agent/runtime.mjs`：plan/build 阶段、审批边界、依赖调度、事件流、验证和修复。
- `agent/tools.mjs`：受控工具、schema 规范化、`appType` 识别、产物与行为校验。
- `agent/model.mjs`：DeepSeek V4 Pro 思考模式调用与 JSON 解析，由 Node 和 Worker 复用；为推理和结构化结果提高 token 上限与超时时间。
- `server.mjs` / `worker.mjs`：本地与 Cloudflare HTTP 边界。
- `site/js/planner.js`：工作区生命周期、每轮上下文清理和本地降级计划。
- `site/js/interactive.js`：计算器与贪吃蛇的纯 reducer 和状态规范化。
- `site/js/code-workspace.js`：源码规范化、编辑校验、CSS 限制、revision 与回滚。
- `site/js/auth.js`：登录态和云端工作区同步。
- `backend/auth.mjs`：PBKDF2 密码散列、HttpOnly 会话与 KV/内存存储适配。
- `site/js/app.js`：事件流消费、Reviewer 渲染、代码编辑、交互调度和完整预览模式。

## 交互产物模型

```text
模型 artifact（紧凑 schema / patch）
  ↓ normalizeArtifact(intent, previousPreview)
{
  appType: "generic" | "calculator" | "snake",
  title, subtitle, themeId, sections, files, ...
}
  ↓ createSourceFiles
src/App.jsx + src/styles.css + src/app.config.json
  ↓
generic → navigation / CTA / list / design commands
calculator → calculator reducer → display / history
snake → snake reducer + timer → board / direction / score
  ↓
previewInteractions[workspaceId] + versions[revision]
  ↓
账号本地缓存 + Cloudflare KV → 主 Viewer / 完整预览复用
```

模型负责识别用户想做什么和应用内容；真正的按钮行为由确定性 reducer 执行。这样可以真实操作，也能对计算结果、碰撞、得分和状态迁移写单元测试。`appType` 是受控枚举，未知值会回落到 `generic`。

模型不再重复输出长篇源码 JSON。它生成经过约束的应用 schema 或增量 patch，runtime 再从验证通过的结果生成三份真实、可编辑的源码。这样仍保留模型决定的产品结构和内容，同时避免输出截断，并让 Code、Preview 和 revision 使用同一来源。

## 增量修改协议

增量修改只继承稳定产物，不继承上一轮临时决策：

`artifactRevision + currentPreview → patch(set + sectionOps) → candidate 验证 → 原子提交`

- 每个 section 有稳定 ID，更新和删除必须指定 ID，避免整页覆盖。
- 运行前记录 `baseRevision`；返回时基线不一致则拒绝候选产物。
- 候选产物先做 schema、话题关联、行为、增量范围和版本验证；只有通过才替换 Preview。
- 模型 patch 失败时，修改任务可回落到确定性最小 patch；仍失败则保留 last-known-good。

1. 用户在 ready 工作区继续输入，`hasBuiltArtifact` 使新一轮识别为修改候选。
2. 提交时清空旧 `intent`、`runtimePlan`、runtime events 和运行状态。
3. plan 阶段只发送已有 `preview`，重新做意图识别和计划。
4. 用户批准后，build 阶段发送本轮 `intent`、本轮 plan 和已有 preview。
5. `normalizeArtifact` 优先采用本轮显式领域；修改未显式更换领域时保留原 `appType`、`themeId` 和标题风格。
6. 构建期间禁用输入，避免并发请求互相覆盖工作区。

## Code → Preview 闭环

1. Reviewer 切到 `Code`，从文件树选择 `src/App.jsx`、`src/styles.css` 或 `src/app.config.json`。
2. 用户直接修改文本；未应用状态只存在内存草稿，不覆盖 last-known-good。
3. Apply 解析 App 配置标记或 JSON，拒绝非法结构；CSS 额外拒绝 `@import`、外部 URL、expression 等主动内容。
4. 三文件互相同步，创建新的 revision 快照，再运行 Preview schema、源码一致性、行为连接和 DOM 验证。
5. 失败不覆盖原版本；Ask Agent 把当前文件带回对话；Rollback 从历史快照恢复并生成新的 revision。

## 登录与持久化

```text
注册 / 登录
  ↓ 随机 salt + AUTH_PEPPER HMAC + HttpOnly SameSite Cookie
session → user
  ↓
GET/PUT /api/workspaces
  ↓
Cloudflare KV: user / session / workspace snapshot
  ↕
按 userId 隔离的 localStorage 快速缓存
```

Node 本地服务使用同一接口和进程内存适配器；Cloudflare 通过 `ATOMS_DATA` KV binding 持久化。Agent API 与工作区 API 都要求有效会话，前端不会接触 DeepSeek Key。

这套协议解决了“二次修改复用旧计划”“修改后应用退化为通用静态页面”和“连续点击触发并发构建”三个问题。

## App Viewer 与滚动边界

- 工作台高度使用 `100dvh`、`minmax(0, 1fr)` 和 `min-height: 0`，不依赖嵌套 `calc()`。
- 对话区 `.message-stream`、应用区 `.design-preview`、Console 和 Files 分别拥有独立纵向滚动容器。
- 完整预览使用 `?preview=<workspaceId>` 进入同源 preview-only 布局，复用交互状态，不再生成静态 blob 快照。
- Design、Library、Theme、Preview、Code 和设备尺寸都更新同一份工作区状态；刷新只重新渲染，不清除交互或生成产物，重置保留为独立操作。
- Preview 验证结果和交互反馈会保存到 workspace，并随下一轮 plan/build 返回 Agent。

## 稳定性与安全

- 模型输出必须经过 schema 白名单、类型规范化、长度限制和行为验证。
- 所有工具是本地受控函数，不接受模型提供的 shell 命令或任意代码。
- 计划批准是执行边界；模型 JSON 损坏时使用确定性计划或安全产物继续验证，验证失败最多自动修复一次。
- Cloudflare 密码校验使用随机 salt 与独立 `AUTH_PEPPER` 的 HMAC 摘要，避免免费 Worker 的 10 ms CPU 预算被高迭代 PBKDF2耗尽；本地未配置 pepper 时保留 PBKDF2。
- Session 只写入 HttpOnly Cookie；工作区按 userId 分区。
- 浏览器请求限定同源、JSON、32 KB；上下文字段逐项清洗；Agent 流使用 NDJSON。
- Cloudflare 对 Agent API 按 IP 限流 20 次/分钟；健康检查不暴露 Secret。
- DeepSeek API Key 仅保存在本地 `.env` 或 Cloudflare Secret，永不进入前端 bundle、Git 或 runtime 事件。

## 能力边界

本项目复刻 Atoms 的创作体验和 Agent 核心，不是通用云 IDE。当前已有基础账号、云端工作区、源码 revision 和回滚，但没有任意代码沙箱、依赖安装、邮箱验证、OAuth 连接器、多人协作、分支合并和生成应用的独立部署。计算器与贪吃蛇用于证明“意图识别 → 领域行为 → 可验证交互”的扩展方式；继续扩展任意代码能力时应先建设真正的隔离沙箱，而不是直接在前端执行模型输出。
