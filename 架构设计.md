# Atoms Demo 架构设计

## 产品主线

界面围绕“对话 + App Viewer”组织，不要求用户先选择模式。项目列表、Console、Files、Design 和 Code 是辅助检查面；意图、澄清、计划、工具执行、验证与增量修改都回到同一条对话时间线。

## Agent runtime

```text
用户输入
  ↓
识别 intent + domain ── 关键输入不足 ──→ 结构化澄清 ──→ 恢复规划
  ↓
选择必要专家与受控工具，生成依赖计划
  ↓
人工批准
  ↓
执行 tool → 写入共享 artifact → 发出流式事件
  ↓
规范化 UI schema + appType
  ↓
validate ── 失败 ──→ 单次 repair ──→ 再验证
  ↓
App Viewer + Working Process
```

模块职责：

- `agent/intent.mjs`：意图、领域、置信度、特征和澄清判断。
- `agent/prompts.mjs`：模型输出协议；约束修改任务以已有 preview 为基线。
- `agent/runtime.mjs`：plan/build 阶段、审批边界、依赖调度、事件流、验证和修复。
- `agent/tools.mjs`：受控工具、schema 规范化、`appType` 识别、产物与行为校验。
- `agent/model.mjs`：DeepSeek Flash 调用与 JSON 解析，由 Node 和 Worker 复用。
- `server.mjs` / `worker.mjs`：本地与 Cloudflare HTTP 边界。
- `site/js/planner.js`：工作区生命周期、每轮上下文清理和本地降级计划。
- `site/js/interactive.js`：计算器与贪吃蛇的纯 reducer 和状态规范化。
- `site/js/app.js`：事件流消费、Viewer 渲染、交互调度和完整预览模式。

## 交互产物模型

```text
模型 artifact
  ↓ normalizeArtifact(intent, previousPreview)
{
  appType: "generic" | "calculator" | "snake",
  title, subtitle, themeId, sections, files, ...
}
  ↓
generic → navigation / CTA / list / design commands
calculator → calculator reducer → display / history
snake → snake reducer + timer → board / direction / score
  ↓
previewInteractions[workspaceId]
  ↓
localStorage → 主 Viewer / 完整预览复用
```

模型负责识别用户想做什么和应用内容；真正的按钮行为由确定性 reducer 执行。这样可以真实操作，也能对计算结果、碰撞、得分和状态迁移写单元测试。`appType` 是受控枚举，未知值会回落到 `generic`。

## 增量修改协议

增量修改只继承稳定产物，不继承上一轮临时决策：

1. 用户在 ready 工作区继续输入，`hasBuiltArtifact` 使新一轮识别为修改候选。
2. 提交时清空旧 `intent`、`runtimePlan`、runtime events 和运行状态。
3. plan 阶段只发送已有 `preview`，重新做意图识别和计划。
4. 用户批准后，build 阶段发送本轮 `intent`、本轮 plan 和已有 preview。
5. `normalizeArtifact` 优先采用本轮显式领域；修改未显式更换领域时保留原 `appType`、`themeId` 和标题风格。
6. 构建期间禁用输入，避免并发请求互相覆盖工作区。

这套协议解决了“二次修改复用旧计划”“修改后应用退化为通用静态页面”和“连续点击触发并发构建”三个问题。

## App Viewer 与滚动边界

- 工作台高度使用 `100dvh`、`minmax(0, 1fr)` 和 `min-height: 0`，不依赖嵌套 `calc()`。
- 对话区 `.message-stream`、应用区 `.design-preview`、Console 和 Files 分别拥有独立纵向滚动容器。
- 完整预览使用 `?preview=<workspaceId>` 进入同源 preview-only 布局，复用交互状态，不再生成静态 blob 快照。
- Design、Library、Theme、Preview、Code 和设备尺寸都更新同一份工作区状态；刷新预览只清除临时交互，不清除生成产物。

## 稳定性与安全

- 模型输出必须经过 schema 白名单、类型规范化、长度限制和行为验证。
- 所有工具是本地受控函数，不接受模型提供的 shell 命令或任意代码。
- 计划批准是执行边界；验证失败最多自动修复一次，仍失败则向用户返回明确错误。
- 浏览器请求限定同源、JSON、32 KB；上下文字段逐项清洗；Agent 流使用 NDJSON。
- Cloudflare 对 Agent API 按 IP 限流 20 次/分钟；健康检查不暴露 Secret。
- DeepSeek API Key 仅保存在本地 `.env` 或 Cloudflare Secret，永不进入前端 bundle、Git 或 runtime 事件。

## 能力边界

本项目复刻 Atoms 的创作体验和 Agent 核心，不是云 IDE。当前没有任意代码沙箱、依赖安装、真实数据库/Auth、OAuth 连接器、多人协作、完整版本回放和生成应用的独立部署。计算器与贪吃蛇用于证明“意图识别 → 领域行为 → 可验证交互”的扩展方式；继续扩展时应新增受控领域 runtime，或先建设真正的隔离沙箱，而不是直接在前端执行模型输出。
