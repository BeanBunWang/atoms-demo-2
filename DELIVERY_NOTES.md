# Atoms 产品复刻作业：交付说明

## 总体结论

本项目已经完成 Atoms 核心应用构建体验的可运行复刻，达到“可演示、可操作、可评审”的作业交付标准。

用户从一句需求开始，系统先识别任务意图、目标用户、领域和约束，再由 Mike 动态选择必要专家并生成计划。计划批准后，专家依次执行研究、产品定义、设计系统、应用组合、数据分析等受控工具，上游产物会进入下一步共享上下文。最终产物经过结构、话题关联、布局差异和文件完整性验证；失败时自动修正一次。

这不是“固定流程播放”：不同输入会得到不同意图、专家组合、工具链、页面模板、模块结构、文案和数据。例如宠物健康应用会生成疫苗时间线与宠物档案，销售分析需求会生成漏斗、重点商机和风险模块。

## 评估结果

- **完成度**：核心 Agent 工作流和应用查看器完整；真实代码沙箱、云资源和生产连接器不在本次范围。
- **工程思维**：模型负责语义决策，schema 与受控工具保证可运行性；密钥只在服务端；人工审批和单次重规划限制失控风险。
- **用户体验**：输入、审批、运行观察、预览、编辑和发布状态在一个工作区连续完成；placeholder、空态、设备预览和本地保存均可用。
- **创新性**：同一 runtime 同时支持任务意图路由、专家共享上下文和话题化 UI schema，能解释“为什么由谁做、调用了什么、结果如何验证”。
- **可交付性**：源码、GitHub Pages、本地真实模型启动、测试和能力边界均已提供。

## 当前边界

- 使用 `npm start` 本地运行时调用真实 DeepSeek V4 Flash。
- GitHub Pages 无法安全保存服务端密钥，因此在线版本自动使用本地规则降级，并在页面明确显示运行模式。
- 专家角色由同一 DeepSeek Flash 模型加不同阶段上下文驱动，并非 8 个常驻独立模型进程。
- 附件会读取文本内容进入上下文；GitHub 连接器当前只提供演示上下文，不访问私人仓库。
- Deep Research 是模型上下文研究，不等同于带实时引用的联网检索。
- Publish 只完成产品交互闭环，不会创建真实域名、数据库或云部署。

## 后续扩展优先级

1. **P0：隔离代码沙箱** — 真正安装依赖、运行生成代码，并把编译和测试错误送回 runtime 修复。
2. **P0：在线模型代理与预览发布** — 用 Cloudflare Pages Functions/Workers 安全托管模型代理和临时预览环境。
3. **P1：真实连接器** — 接入 GitHub、Figma、Supabase 等 OAuth 上下文，并增加授权和审计。
4. **P1：独立专家上下文** — 为各专家提供独立消息历史、权限和持久产物，而不是共享单一模型调用层。
5. **P2：Race Mode 完整对比** — 同时生成两个可切换版本，以明确指标选出胜者。
