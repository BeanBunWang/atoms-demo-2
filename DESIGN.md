# Atoms Demo 设计说明

## 产品定位

这不是一个任务看板，而是一个“AI 应用构建工作台”。用户在同一界面完成需求沟通、计划审批、构建观察、应用查看、可视编辑和发布，减少不同工具之间的上下文切换。

## 信息架构

- 左侧全局导航：项目、资源入口和本地工作区。
- 对话区：需求消息、实施计划、模式选择和后续修改指令。
- App Viewer：Preview / Code、设备视口、Design mode 和独立预览。
- 活动区：智能体状态、终端日志和文件树。

## 关键取舍

- 使用 DeepSeek V4 Flash 生成真实计划和预览规格，再由确定性状态机推进智能体接力，兼顾模型能力与演示稳定性。
- 预览是结构化模板，不执行任意用户代码，避免在纯静态站点中伪造不安全沙箱。
- 计划必须审批后才能构建，构建完成后才能发布，动作与系统状态保持一致。
- placeholder 由原生属性和显式 `.has-value` 状态双重约束，输入后立即隐藏。

## 视觉系统

- 白色/浅灰工具界面，蓝色只用于主操作和活动状态。
- 4px 基础间距，界面圆角 6–10px，重点容器使用细边框与轻阴影。
- 生成应用预览使用 Georgia 建立产品感，与工具界面的系统无衬线区分。
- 只使用状态过渡和消息进入动画，并尊重 `prefers-reduced-motion`。

## 能力边界

模型代理位于 `server.mjs`，状态机与预览模板位于 `site/js/planner.js`。API Key 只由服务端读取；浏览器端只接收经过结构校验的产品计划和预览规格。若升级为真实产品，应继续增加隔离构建环境和发布提供商，不应把 API Key 或任意代码执行能力放入浏览器端。
