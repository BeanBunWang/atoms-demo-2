# Atoms Demo 重构结论

## 问题判断

旧版只是在浏览器里按固定顺序播放智能体状态，无法体现 Atoms 的意图理解、专家路由、共享上下文、人工审批与交付验证。当前版本已经替换为服务端两段式 Agent runtime，并保留 App Viewer、项目状态和可视编辑能力。

## 行为回归基线

- 用户输入内容后，原生 placeholder 和任何空态提示都必须隐藏。
- 刷新页面后保留工作区状态；损坏或旧版存储不能阻断启动。
- 计划审批前不进入构建；审批后只执行由意图和计划选中的专家与工具。
- 不同话题必须改变页面模板、模块、文案和数据，而不是只替换标题。
- runtime 必须暴露意图、决策、工具输出、验证结果和重规划事件。
- 预览、代码、文件、控制台、设备尺寸和设计模式均有可见反馈。
- 发布动作只模拟交付结果，不冒充真实云部署。

## 已完成范围

1. 用 Atoms 风格的侧栏、对话区、App Viewer、终端/文件区替换项目看板。
2. 增加服务端意图识别、动态计划、受控工具、共享上下文、验证和单次重规划。
3. 使用本地静态资产，避免运行时依赖第三方图片。
4. 保留原生 HTML/CSS/JavaScript、localStorage、PWA 和 GitHub Pages。
5. 补充 runtime、模型代理、存储、输入空态和页面结构回归测试。

## 明确不做

- 已接入本地 DeepSeek V4 Flash，但不接入代码沙箱、数据库或云部署凭据。
- 不伪造可运行的任意用户生成代码；预览由确定性模板驱动。
- 不引入前端框架或新依赖，控制 Demo 的部署和维护复杂度。
